# Requirements — ts-to-sv-compiler-core

Requirements use the SHALL format. Each requirement is keyed REQ-COMP-NNN.
Requirements are normative and binding on all Compiler Agent implementations.

---

## General Compilation

REQ-COMP-001: The compiler SHALL accept TypeScript source as input and produce
IEEE 1800-2017 SystemVerilog as output.

REQ-COMP-002: The compiler SHALL emit a header comment `// Generated by ts2v - TypeScript
to SystemVerilog compiler` as the first line of every output file.

REQ-COMP-003: The compiler SHALL emit `` `timescale 1ns / 1ps `` and
`` `default_nettype none `` / `` `default_nettype wire `` guards in every output file.

REQ-COMP-004: The compiler SHALL report all errors with a source location (file, line,
column) and an error code prefix (TS2V-NNNN).

REQ-COMP-005: The compiler SHALL NOT emit `wire` declarations; all signals must use `logic`.

REQ-COMP-006: The compiler SHALL NOT emit `input wire logic`; input ports must use `input logic`.

REQ-COMP-007: The compiler SHALL NOT infer latches; all combinational outputs must be fully
assigned in every branch.

REQ-COMP-008: After any change to `packages/core/` source, the build (`bun run build`) MUST
complete before compiled examples reflect the change.

---

## Functional Compiler

REQ-COMP-010: The functional compiler SHALL accept `function` declarations and map each one
to an independent SV module.

REQ-COMP-011: The functional compiler SHALL map TypeScript `number` to `logic [31:0]` and
`boolean` to `logic`.

REQ-COMP-012: The functional compiler SHALL map function parameters to `input logic` ports
and return values to an `output logic` port named `result`.

REQ-COMP-013: The functional compiler SHALL support all arithmetic, bitwise, comparison, and
shift operators including `>>>` (arithmetic right shift per IEEE 1800-2017 §11.4.10).

REQ-COMP-014: The functional compiler SHALL map `if/else` control flow to ternary mux chains
in `assign` statements (no latches; no `always_comb`).

REQ-COMP-015: The functional compiler SHALL reject `class`, `interface`, `=>` (arrow
functions), `async`, `for`, `while`, and `string` with error code TS2V-2000 and a
descriptive message.

REQ-COMP-016: The functional compiler SHALL map hex literals (e.g. `0xFF`) to sized SV hex
literals (e.g. `32'hFF`) and binary literals (e.g. `0b1010`) to sized binary literals
(e.g. `4'b1010`).

REQ-COMP-017: The functional compiler SHALL strip underscore separators in numeric literals
(e.g. `10_000` -> `32'd10000`).

---

## Class Compiler

REQ-COMP-020: The class compiler SHALL accept TypeScript classes decorated with `@Module`
and emit one SV module per class.

REQ-COMP-021: The class compiler SHALL map `@Input` properties to `input logic` ports and
`@Output` properties to `output logic` ports.

REQ-COMP-022: The class compiler SHALL map `@Sequential` methods to `always_ff @(posedge clk
or negedge rst_n)` blocks with non-blocking assignments (`<=`).

REQ-COMP-023: The class compiler SHALL map `@Combinational` methods to `always_comb` blocks
with blocking assignments (`=`).

REQ-COMP-024: The class compiler SHALL inject an async active-low reset block into
`always_ff` unless `@ModuleConfig('resetSignal: "no_rst"')` is present.

REQ-COMP-025: The class compiler SHALL support `switch/case/default/break` and emit
`case(...) endcase` in the generated SV.

REQ-COMP-026: The class compiler SHALL map top-level TypeScript `enum` declarations to
`typedef enum logic [N:0] { ... }` with automatically computed bit-width.

REQ-COMP-027: The class compiler SHALL promote method-local `let`/`const` variables that are
assigned across multiple branches to module-level `logic` registers.

REQ-COMP-028: The class compiler SHALL support private helper methods and inline their bodies
at call sites. No SV function calls shall appear in generated output.

REQ-COMP-029: The class compiler SHALL support `@Submodule` and emit named-port instantiation
of child modules, auto-wiring ports by name.

REQ-COMP-030: The class compiler SHALL support early `return` statements by converting them
to equivalent `if/else` chains.

REQ-COMP-031: The class compiler SHALL support `Logic<N>`, `Bit`, `UintN`, `UIntN`, `Logic<N>[]` with explicit size, and `LogicArray<W, SIZE>` type annotations. `Logic<N>` and its aliases map to `logic [N-1:0]`. `LogicArray<W, SIZE>` is equivalent to `Logic<W>[]` with array size `SIZE` and SHALL produce `logic [W-1:0] name [0:SIZE-1]` in SV output.

REQ-COMP-032: The class compiler SHALL support complex arithmetic expressions such as
`((a * 3) << 16) | ((b - c) * 3)` and emit correct SV operator precedence.

REQ-COMP-033: The class compiler SHALL support `@Assert(condition)` and emit an SVA
`assert property` statement.

REQ-COMP-034: The class compiler SHALL support `@ModuleConfig` options: `resetSignal`,
`active_high`, and `synchronous`.

---

## Multi-File / Directory Compilation

REQ-COMP-040: When given a directory path, the compiler SHALL collect all `.ts` files,
strip import/export statements, concatenate the sources, and compile as a single design.

REQ-COMP-041: The compiler SHALL produce a single `.sv` output file containing all modules
for a directory compilation.

REQ-COMP-042: The compiler SHALL NOT attempt to compile client scripts (files importing
Node/Bun APIs) if they reside in a `client/` subdirectory that is not the compilation target.

---

## LogicArray Type Support

REQ-COMP-035: The class compiler's `parseTypeSpec` function SHALL recognise the identifier `LogicArray` and parse its two generic arguments `<W, SIZE>` into a `PropertyAST` node with `is_array: true`, `bit_width: W`, `array_size: SIZE`.

REQ-COMP-036: The class-module emitter SHALL check that `array_size > 0` for every `is_array: true` property before emitting. If `array_size` is zero or missing, the emitter SHALL emit a `CompilerError` with code `ERROR_ARRAY_SIZE_REQUIRED` and abort emission of that module.

---

## Parameterised Modules

REQ-COMP-037: The class compiler SHALL recognise the `@Param` property decorator and record the decorated property as a module-level SV `parameter` in the `ClassModuleAST`. The property SHALL NOT be emitted as a port, localparam, or internal signal.

REQ-COMP-038: When a submodule class has `@Param`-decorated properties, the parent module's submodule instantiation SHALL emit `#(.NAME(VALUE))` parameter overrides using the values provided at the point of submodule construction.

REQ-COMP-039: Parameter names SHALL be unique within a module and SHALL NOT collide with port or internal signal names. A name collision SHALL cause the compiler to emit a `CompilerError` and fail compilation.

---

## Bit-Slice Intrinsics

REQ-COMP-060: The class compiler SHALL recognise calls of the form `Bits.slice(signal, msb, lsb)` as a compiler intrinsic and translate them to the SystemVerilog part-select expression `signal[msb:lsb]`.

REQ-COMP-061: The class compiler SHALL recognise calls of the form `Bits.bit(signal, i)` as a compiler intrinsic and translate them to the SV single-bit select `signal[i]`.

REQ-COMP-062: The identifier `Bits` when used as a qualifier for `slice` or `bit` SHALL be treated as a reserved compiler namespace. The compiler SHALL emit a non-fatal warning if the user declares a class property or local variable named `Bits`.

---

## Error Handling

REQ-COMP-050: Lexer errors SHALL carry error code prefix TS2V-1000.
REQ-COMP-051: Parser errors SHALL carry error code prefix TS2V-2000.
REQ-COMP-052: Type checker errors SHALL carry error code prefix TS2V-3000.
REQ-COMP-053: Code generator errors SHALL carry error code prefix TS2V-4000.
REQ-COMP-054: All error messages SHALL include line number and column position.
REQ-COMP-055: The compiler SHALL exit with a non-zero status code when any error occurs.
