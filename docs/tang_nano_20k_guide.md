# ts2v v1.0.0 — Usage Guide

## Deploying to Sipeed Tang Nano 20K (GW2AR-18C)

**ts2v** transpiles TypeScript classes into IEEE 1800-2017 compliant SystemVerilog.
This guide walks through writing hardware in TypeScript, compiling it, and getting
it running on real silicon — specifically the Tang Nano 20K.

---

## 1. Quick Start

### Install

```bash
git clone <repo> && cd ts2v
npm install
```

### Write Hardware

Create a file `my_counter.ts`:

```typescript
@Module
class Counter extends Module {
  @Input  clk:   Logic<1>;
  @Input  rst_n: Logic<1>;
  @Output count: Logic<8> = 0;

  @Sequential(clk)
  tick() {
    this.count++;
  }
}
```

### Compile

```bash
npx ts-node src/cli.ts compile my_counter.ts -o build/counter.sv
```

Output:

```systemverilog
// Generated by ts2v v1.0.0 - IEEE 1800-2017 Compliant SystemVerilog
`timescale 1ns / 1ps
`default_nettype none

module Counter (
    input  wire logic  clk,
    input  wire logic  rst_n,
    output      logic [7:0] count
);

    // Sequential Logic (tick)
    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            count <= 8'd0;      // ← context-aware: 8-bit target gets 8'd0
        end else begin
            count <= count + 1;
        end
    end

endmodule

`default_nettype wire
```

## 2. Language Reference

### Decorators

| Decorator | Purpose | Example |
|-----------|---------|---------|
| `@Module` | Declares a hardware module | `@Module class ALU extends Module { ... }` |
| `@Input` | Input port | `@Input data: Logic<8>;` |
| `@Output` | Output port (with reset value) | `@Output result: Logic<16> = 0;` |
| `@Sequential(clk)` | Clocked logic (always_ff) | Method body uses `<=` |
| `@Combinational` | Combinational logic (always_comb) | Method body uses `=` |
| `@Submodule` | Instantiate child module | `@Submodule child: PwmCore = new PwmCore();` |
| `@Assert(cond)` | SVA assertion | `@Assert(this.count < 15)` |

### Types

| TypeScript | Bit Width | SystemVerilog |
|------------|-----------|---------------|
| `Logic<1>` | 1 bit | `logic` |
| `Logic<4>` | 4 bits | `logic [3:0]` |
| `Logic<8>` | 8 bits | `logic [7:0]` |
| `Logic<32>` | 32 bits | `logic [31:0]` |
| `boolean` / `Bit` | 1 bit | `logic` |

### Operators

| TypeScript | SystemVerilog | Notes |
|------------|--------------|-------|
| `===` | `==` | Equality |
| `!==` | `!=` | Inequality |
| `&` | `&` | Bitwise AND |
| `\|` | `\|` | Bitwise OR |
| `^` | `^` | Bitwise XOR |
| `~` | `~` | Bitwise NOT |
| `<<` / `>>` | `<<` / `>>` | Shift |
| `0xFF` | `8'hFF` | Hex literal (auto-sized) |

### Context-Aware Bit-Width Emission (v1.0.0)

The compiler inspects the **target** of every assignment and sizes bare
numeric literals to match:

```typescript
@Output flag: Logic<1> = 0;    // Reset: 1'b0    (not 32'd0!)
@Output data: Logic<8> = 0;    // Reset: 8'd0
@Output addr: Logic<4> = 0;    // Reset: 4'd0

this.flag = 1;                 // Emits: flag <= 1'b1
this.data = 255;               // Emits: data <= 8'd255
this.addr = 5;                 // Emits: addr <= 4'd5
```

Complex expressions (e.g. `this.count + 1`) pass through unsized — only
bare decimal literals get the width prefix.

### Module Hierarchy (v1.0.0)

Instantiate child modules with `@Submodule`:

```typescript
@Module
class PwmCore extends Module {
  @Input  clk:     Logic<1>;
  @Input  rst_n:   Logic<1>;
  @Input  duty:    Logic<8>;
  @Output pwm_out: Logic<1> = 0;
  // ...
}

@Module
class Top extends Module {
  @Input  clk:   Logic<1>;
  @Input  rst_n: Logic<1>;
  @Output led:   Logic<1> = 0;

  @Submodule pwm: PwmCore = new PwmCore();
  // ...
}
```

Generates:

```systemverilog
PwmCore pwm (
    .clk(clk),
    .rst_n(rst_n),
    .duty(duty),
    .pwm_out(/* unconnected */)
);
```

The compiler uses a two-pass approach: Pass 1 builds a module signature
table, Pass 2 auto-binds ports by name.

### SVA Assertions (v1.0.0)

Class-level (property assertion):

```typescript
@Assert(this.count < 15)
```

Generates:

```systemverilog
assert_0: assert property (@(posedge clk) count < 15);
```

Inline (immediate assertion inside a method):

```typescript
Assert(this.val < 200);
```

Generates:

```systemverilog
assert (val < 200);
```

## 4. Complete Example: LED Blinker

### TypeScript Source

`examples/tang_nano_20k/tang_nano_20k_blinker.ts`:

```typescript
enum BlinkPattern { ALL_ON, ALTERNATE, CHASE, BREATHE }

@Module
class Blinker extends Module {
  @Input  clk:   Logic<1>;
  @Input  rst_n: Logic<1>;
  @Input  btn:   Logic<1>;
  @Output led:   Logic<6> = 0;
  private counter: Logic<24> = 0;
  private pattern_sel: Logic<2> = 0;
  private chase_pos: Logic<3> = 0;

  @Assert(this.chase_pos < 6)

  @Sequential(clk)
  tick() {
    this.counter++;
    if (this.counter === 0) {
      this.chase_pos++;
      if (this.chase_pos === 6) {
        this.chase_pos = 0;
      }
    }
    if (this.btn === 0) {
      this.pattern_sel = this.pattern_sel + 1;
    }
  }

  @Combinational
  drive() {
    switch (this.pattern_sel) {
      case BlinkPattern.ALL_ON:
        this.led = 0x3F;
        break;
      case BlinkPattern.ALTERNATE:
        if (this.counter[23] === 1) {
          this.led = 0x2A;
        } else {
          this.led = 0x15;
        }
        break;
      case BlinkPattern.CHASE:
        this.led = 1 << this.chase_pos;
        break;
      default:
        this.led = 0;
        break;
    }
  }
}
```

### Compiled SystemVerilog Output

```systemverilog
// Generated by ts2v v1.0.0 - IEEE 1800-2017 Compliant SystemVerilog
`timescale 1ns / 1ps
`default_nettype none

typedef enum logic [1:0] {
    ALL_ON = 2'd0,
    ALTERNATE = 2'd1,
    CHASE = 2'd2,
    BREATHE = 2'd3
} BlinkPattern;

module Blinker (
    input  wire logic  clk,
    input  wire logic  rst_n,
    input  wire logic  btn,
    output      logic [5:0] led
);

    // Internal signals
    logic [23:0] counter;
    logic [1:0] pattern_sel;
    logic [2:0] chase_pos;

    // Sequential Logic (tick)
    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            led <= 6'd0;
            counter <= 24'd0;
            pattern_sel <= 2'd0;
            chase_pos <= 3'd0;
        end else begin
            counter <= counter + 1;
            if (counter == 0) begin
                chase_pos <= chase_pos + 1;
                if (chase_pos == 6) begin
                    chase_pos <= 3'd0;
                end
            end
            if (btn == 0) begin
                pattern_sel <= pattern_sel + 1;
            end
        end
    end

    // Combinational Logic (drive)
    always_comb begin
        case (pattern_sel)
            ALL_ON: begin
                led = 8'h3F;
            end
            ALTERNATE: begin
                if (counter[23] == 1) begin
                    led = 8'h2A;
                end else begin
                    led = 8'h15;
                end
            end
            CHASE: begin
                led = 1 << chase_pos;
            end
            default: begin
                led = 6'd0;
            end
        endcase
    end

    // SystemVerilog Assertions
    assert_0: assert property (@(posedge clk) chase_pos<6);

endmodule

`default_nettype wire
```

## 5. Deploying to Hardware

### Prerequisites

- Sipeed Tang Nano 20K board
- Gowin EDA (free Education edition) OR open-source toolchain (Yosys + nextpnr-gowin + openFPGALoader)
- USB-C cable

### Option A: Gowin EDA (GUI)

1. Create new project → Device: **GW2AR-LV18QN88C8/I7** (filter: GW2AR-18C)
2. Add source: `build/tang_nano_20k_blinker.sv`
3. Add constraints: `constraints/tang_nano_20k.cst`
4. Set **Blinker** as top module
5. Synthesize → Place & Route → Program Device

### Option B: Open-Source Toolchain

```bash
# Synthesize
yosys -p "
  read_verilog -sv build/tang_nano_20k_blinker.sv;
  synth_gowin -top Blinker -json build/blinker.json
"

# Place & Route
nextpnr-gowin \
  --json build/blinker.json \
  --write build/blinker_pnr.json \
  --device GW2AR-LV18QN88C8/I7 \
  --family GW2AR-18C \
  --cst constraints/tang_nano_20k.cst

# Pack bitstream
gowin_pack -d GW2AR-18C -o build/blinker.fs build/blinker_pnr.json

# Flash
openFPGALoader -b tangnano20k build/blinker.fs
```

### Expected Behavior

- On power-up: all 6 LEDs solid ON (pattern ALL_ON)
- Press button S2 (pin 87): cycles through patterns
  - ALL_ON → ALTERNATE (LEDs toggle at ~1.6 Hz) → CHASE (single LED sweeps) → OFF → repeat
- The 24-bit counter at 27 MHz wraps every ~0.62 seconds, creating visible blink timing

### Troubleshooting

- **LEDs don't light**: Tang Nano 20K LEDs are active-low. Add `assign led = ~led_internal;` in a wrapper if needed.
- **Button doesn't cycle**: Button S2 active-low with pull-up. The code checks `btn === 0` — press and hold briefly.
- **Programming fails**: Ensure BL616 is in JTAG mode (default). Try `openFPGALoader -b tangnano20k -f build/blinker.fs` for flash.

## 6. Adding Your Own Board

1. Create `boards/my_board.board.json` following the schema above
2. Set `vendor` to `gowin`, `xilinx`, `intel`, or `lattice`
3. Map your clock, reset, and I/O pins
4. Run: `npx ts-node src/cli.ts constrain boards/my_board.board.json`
5. The constraint generator outputs the correct format (.cst, .xdc, .qsf, or .lpf)
